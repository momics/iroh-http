Stream files over httpi://

          

            Loads peer-hosted media straight into native <audio>
            and <img> elements through the httpi://
            scheme handler — no JavaScript bridging, no base64. The webview pulls
            each file (with Range seeking for audio) directly through
            iroh-http-core over QUIC.
          

          

            Start a host with the Deno example:
            cd examples/deno && deno task host. It serves
            audio.wav and image.png and prints its node
            ID — paste that below.
          

          

            Or host them from this node: open the Server
            tab, click Start serving, then press
            Use my node ID below. Streaming from your own ID is
            served in-process via the self-request loopback — no network hop.

    Testing mode
    
          
    
            Opt-in interop harness (#340). While on, this device serves the
            shared HTTP compliance suite and advertises a
            test=1 intent over DNS-SD — including its dialable
            address so peers can dial directly instead of over
            relay. Off by default; never ship enabled.
          
    
          
    
            Exposure: the handler runs on your persisted node
            identity, which is reachable through public relays — not just the
            LAN. As a guard it rejects any caller that isn't this node or a
            discovered test=1 peer (HTTP 403), caps request bodies
            (8 MiB) and limits concurrent fixtures. It is a test affordance,
            not an authenticated service.
